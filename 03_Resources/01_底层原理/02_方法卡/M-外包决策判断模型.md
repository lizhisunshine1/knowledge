# M-外包决策判断模型

## 解决什么问题
判断企业的某个生产环节是否应该外包，避免只看表面单价而忽略整体利润影响。

## 适用场景
- 企业生产能力富余但外部报价更低时
- 销售人员/业务部门有外包决策权时
- 核心能力 vs 非核心能力的剥离判断

## 操作步骤
1. **列出所有成本项**：原材料、人工、设备折旧、管理费等
2. **区分相关成本与不相关成本**：
   - 相关成本：会因外包决策而变化的（原材料、可裁减的人工）
   - 不相关成本：无论做不做都存在的（设备折旧、不可裁减的人工）
3. **计算内部生产成本（仅相关成本）**：只计相关成本的总和
4. **比较**：若代工报价 < 内部相关成本 → 外包划算；反之则自己生产
5. **评估战略风险**：若该环节只有一家供应商能做 → 慎用外包（受制于人）

## 三种可行外包情景
| 情景 | 条件 |
|------|------|
| 分工协作式 | 自己不擅长的零部件，交给专业厂家 |
| 价格优于内部 | 代工价 < 自己的相关成本 |
| 产能已满 | 内部产能饱和，找代工厂补充 |

## 使用条件
- 必须清楚自己的成本结构（料工费明细）
- 需要识别哪些成本是真正"可变的"

## 注意事项
- 销售人员的局部最优 ≠ 公司全局最优（可能损害整体利润）
- 财务数据只是决策的一部分，还需考虑供应链安全、质量、品牌等非财务因素

## 例子
运动鞋厂：每双鞋原材料30元+工资20元+折旧15元=65元成本，代工厂报价60元。表面看外包省5元，但如果工人不能辞退、设备折旧照付，实际只省30元原材料费，却要付60元代工费 → 亏损30元/双。

## 可复用到哪些地方
- SaaS产品的自研 vs 采购第三方API决策
- 咨询项目的外包 vs 自建团队
- 内容创作的AI辅助 vs 纯人工
