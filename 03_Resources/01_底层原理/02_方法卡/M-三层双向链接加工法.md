# M-三层双向链接加工法

> 来自：[[99_System/08_渐进式披露规则]]
> 原文：实验验证文章 [[思维导图：在全国大停电的古巴，我见到了一位发光的年轻人]]

## 解决什么问题
从原始素材到知识卡片的过程中，**摘要太粗找不到原文、原文太细记不住结构**。需要在原文、思维导图、卡片之间建立可导航的双向链接，让知识从"静态存储"变成"可调用网络"。

## 核心结构

```
原文 (inbox/)
  │  ^sX-Y 块引用锚点 → 精确到段落
  │  反向链接 → 思维导图 ### 标题锚点
  ▼
思维导图 (abstract/ 标题格式)
  │  ## → 一级分支（章节）
  │  ### → 二级分支（关键节点），heading锚点可被反向跳转
  │  - ➤ → 叶节点，[[原文#^sX-Y]] 链接到原文段落
  │  - 🃏 → 叶节点，[[卡片名]] 链接到知识卡片
  ▼
知识卡片 (概念卡/方法卡/案例卡/原则卡)
  > from: [[思维导图#🎯 节点名]]
  > source: [[原文文件]]
```

## 操作步骤

### Step 1：通读原文，标注关键段落

读完整篇文章，识别出哪些段落包含了**核心框架/模型/案例**，给每个段落编号。

### Step 2：给原文添加 `^block-id` 锚点

在每段关键内容的末尾添加纯 ASCII 的块引用 ID：

```markdown
格式：段落内容... ^s<章节>-<序号>
示例：因为极致手工... ^s1-1
```

**命名规则：**
- `^s1-1` = 第1章/第1节，第1个关键点
- `^s1-2` = 第1章/第1节，第2个关键点
- `^s2-1` = 第2章/第1节，第1个关键点

**⚠️ 规则：** 块ID只能用纯 ASCII（`^s1-1`），禁止中文（`^s1-天定` 在 Obsidian 中不可靠）

### Step 3：在原文中添加反向链接到思维导图

紧跟在 `^block-id` 后面：

```markdown
...内容... ^s1-1

→ [[思维导图：文件名#🎯 节点名]]
```

### Step 4：创建思维导图文件（abstract/）

文件路径：`<主题>/abstract/思维导图：<文章名>.md`

格式要求：

```markdown
# 📐 思维导图：文章标题

> **原文位置**：[[inbox/原文文件.md]]

---

## 一、第一章主题

### 🎯 节点1名称（反向链接锚点）
- ➤ 要点描述 [[原文#^s1-1]]

### 🎯 节点2名称
- ➤ 要点描述 [[原文#^s1-2]]
- 🃏 [[关联卡片名]]   ← 若有卡片则加

## 二、第二章主题

### 🎯 节点3名称
- ➤ 要点描述 [[原文#^s2-1]]

---

## 🔗 本篇文章提取的卡片

| 卡片 | 类型 | 说明 |
|------|:----:|------|
| [[卡片路径]] | 🧠 概念 | 摘要 |
```

**关键格式规则：**
- `##` 一级标题 → 思维导图的一级分支
- `###` 一级标题 → 思维导图的二级分支（同时也是反向链接锚点）
- `- ➤` 列表 → 叶节点，放原文链接

### Step 5：更新关联卡片的 `from:` 字段

在每个卡片的文件开头添加：

```markdown
> 来自：[[思维导图：文件名#🎯 节点名]]
> 原文：[[inbox/原文文件.md]]
```

## 输出示例

见测试文章 [[思维导图：在全国大停电的古巴，我见到了一位发光的年轻人]]

## 适用场景

- 所有新入库的素材加工作业
- 原文内容较长（>500字）、有框架可提炼的文章
- 希望后续能快速定位到具体段落的素材

## 可迁移的经验

1. **双向链比单向好**：纯摘要只有"看摘要→搜原文"的单向路径，加上反向链接可以从原文直接回到思维导图
2. **heading锚点 vs block引用**：`###` 标题锚点用于反向跳转（原文→思维导图），`^block-id` 用于正向跳转（思维导图→原文），各司其职
3. **Mindmap NextGen 插件**：用标题格式（`##`、`###`），不要用列表缩进
4. **块ID用纯ASCII**：`^s1-1` 比 `^s1-天定` 更可靠
