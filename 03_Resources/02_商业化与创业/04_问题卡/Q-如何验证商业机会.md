# Q-如何验证商业机会

## 问题定义
已经识别出一个看起来不错的机会，如何用最低成本验证它是否真的可行？验证到哪一步才算"足够"？

## 验证顺序（5 步渐进）

```
写清假设 → 验证需求 → 验证付费 → 验证交付 → 验证扩张
```

### 第 1 步：写清假设（1 小时）

把机会写成一句话：
> **为[谁]，在[什么场景]，解决[什么问题]，带来[什么可量化结果]，靠[什么方式]赚钱。**

如果写不清 → 机会还没想透，不需要往下走。

### 第 2 步：验证需求（1–3 天）

找 5–10 个真实用户做访谈，问：
1. 你现在怎么解决这个问题？
2. 最大的麻烦是什么？
3. 为它付出过什么成本？（时间/金钱/人力）
4. 如果有个方案能解决，你愿意推动吗？
5. 预算从哪里来？

**成功标志：** 大多数人能清晰描述痛点，且有尝试解决的动作。

### 第 3 步：验证付费（1–2 周）

测试用户是否愿意付出真实代价：
- 愿意付定金 / 预付款？
- 愿意深度访谈 1 小时？
- 愿意提供数据和场景做试点？
- 愿意推动内部立项？

**成功标志：** 至少有 1–2 个用户愿意推进到下一步。

### 第 4 步：验证交付（2–4 周）

做最小可行版本，看看能不能真的交付结果：
- 用现有工具拼凑一个 demo
- 手工跑一遍完整流程
- 交付一个最小可用的结果

**成功标志：** 你关注的结果指标（效率/效果/满意度）有明显改善。

### 第 5 步：验证扩张（1–3 月）

一个点能成，再看复制性：
- 能否找到更多同类客户？
- 能否标准化交付？
- 能否形成规模效应？

## 验证到什么程度才算"足够"

| 阶段 | 验证内容 | 投入 | 决定 |
|:----:|----------|:----:|------|
| 阶段 1 | 需求存在 | 几小时 | 继续/放弃 |
| 阶段 2 | 有人愿付 | 1–2 周 | 继续/放弃 |
| 阶段 3 | 能交付结果 | 2–4 周 | 继续/加速/放弃 |
| 阶段 4 | 能复制 | 1–3 月 | All in/放弃 |

## 验证的关键原则

1. **验证的是假设，不是方案**
2. **不要问"你觉得怎么样"，要测试"你愿意做什么"**
3. **用户说"挺好"是最危险的反馈**
4. **最有效的验证是让用户付出真实成本**
