# M-如何发现真实需求

## 解决什么问题
帮助我判断一个看似热闹的方向,究竟有没有真正值得做的需求。

## 适用场景
- 创业选题
- 副业验证
- 产品方向判断
- 内容选题判断

## 操作步骤
1. 先看问题是否高频
2. 再看痛感是否强
3. 再看是否已经有人在付费解决
4. 再看现有方案是否不够好
5. 最后再做小验证

## 需求分类判断

| 类型 | 特征 | 是否值得做 |
|------|------|:----------:|
| **痒点** | 有点用，但不解决也没事 | ❌ 一般不 |
| **痛点** | 明显不舒服，但能忍 | ⚠️ 看情况 |
| **刚需** | 不解决就会出事 | ✅ 重点看 |
| **战略点** | 解决后会改变组织能力/竞争格局 | ✅✅ 优先做 |

## 判断真伪需求的 4 问

1. **用户现在在用别的办法解决吗？**
   - 是 → 需求真实。哪怕方案很笨，说明问题真实存在
   - 否 → 需谨慎，可能只是你的想象

2. **用户已经为这个需求付出过真实成本吗？**
   - 钱、时间、人力、注意力，都算
   - 有 → 强信号。没有 → 弱信号

3. **不解决会带来明显损失吗？**
   - 收入损失 / 效率损失 / 风险增加 / 合规隐患
   - 越靠后，需求越坚实

4. **用户愿意接受付费方案吗？**
   - 愿意付多少钱？预算从哪来？
   - 说不清 → 即使有需求，商业化也难

## 使用条件
- 不要只听单个用户的声音
- 不要只看流量，不看转化

## 注意事项
- 需求表达和真实需求可能不是一回事
- 用户说"想要"不等于愿意付费
- **最危险的信号：** 用户说"挺好"但不愿推进

## 我的版本
高频 + 强痛 + 愿付费 + 方案不完美，才值得继续。
