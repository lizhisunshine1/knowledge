# 刘润：软件，正在成为“耗材”

事情是这样的。

此时此刻，我正在从中国飞厄瓜多尔（南美一个赤道小国）的飞机上。

要飞大约30个小时。

实在太远。几乎正好在地球背面。飞到后面，真是有点崩溃。不知道干什么。醒着很无聊。睡又睡不着。

怎么办？看电影？

不不不。不看电影。我决定，找点比看电影更好玩的事做。

我在手机上打开一个聊天窗口。开始指挥AI … 写程序！

这次“问道全球”很特别。我们不拜访“人”。我们拜访“动物”。

拜访那些，当年在加拉帕戈斯群岛上，“接待过达尔文”的动物。

比如象龟（据说后来启发斯蒂芬·斯皮尔伯格，创造了ET）。比如海鬣蜥（据说后来启发“谁谁谁”，创造了哥斯拉）。

当然还有最早启发达尔文，创立了“进化论”的“达尔文雀”。

所以，我很想写个小软件，用互动的方式，向同行的企业家们介绍清楚，这片群岛真正的主人们。

用什么语言写呢？

我有41年写程序的历史。1985年，邓小平说：学电脑要从娃娃抓起。于是，我被抓去少年宫学BASIC。后来学LOGO。再后来学C。学Pascal。学C++。C#。Java。以及各种脚本语言。到现在的Python。

但今天，我决定以上语言都不用。今天，我要用我最熟悉的语言。

中文！

我把行程PDF文档扔给了AI。然后别上一个领夹麦克风，开始指挥它干活。

“… 写一个智能行程手册 … 要适合手机看 … 要用'时点-地点-看点'的逻辑，串起PDF里所有的素材 … 要有实时天气 … 动态地图 … 要很漂亮 … 要互动 … 要酷 …”

注意：这些要求，都是我在一个聊天窗口里，用错字百出的语音识别，向AI提的。

半小时左右，这个程序就写好了。

看到成果的那一刻 … 我觉得自己这41年的编程 … 白学了！

它的审美的设计，布局的安排，交互的设计，都是专业水平。

“问道全球 · 加拉帕戈斯智能行程手册”，发布在了这里：https://galapagos.manus.space。

有感兴趣的，可以看看。看不了的，我也把截图，放在了留言区。

那一刻，我突然有个感受：软件业的范式，可能从此要从“盖房子”，变为“抽纸巾”了。

随需随取。随取随用。随用随扔。

为什么？

因为当软件的成本，可以便宜到只是“对着AI说几句话”时，那你遇到任何问题，都可以立马“变”出个软件，就和抽出张纸巾一样，用完即扔。

反正便宜。

就像“问道全球 · 加拉帕戈斯智能行程手册”一样，只活7天。

我突然意识到，我们被工具思维困住太久了。

我们家里都有螺丝刀。螺丝刀很有用。非常有用。甚至有用到了，遇到拧不开的螺丝，我们会觉得是这个螺丝有问题。而不是螺丝刀有问题。

你看你看，螺丝齿花了吧。这个螺丝质量不行。

是的。螺丝是有问题。但工具，不就是来解决问题的吗？

螺丝是花了。那就去找一个，正好能卡住这个花齿的螺丝刀来，不就解决问题了吗？

什么？没有这样的螺丝刀？那就定制一个呗。什么？定制太慢，而且太贵？

你看。还是成本的问题。

那如果这个螺丝刀的成本，1分钱定制呢？而且你一眨眼，就定制好了呢？

那么，螺丝刀就会从此从“耐用品”，变为“耗材”。

软件就是这样。

软件，正在变成耗材。

这个表格不会处理？对着AI说几句话，一款专门为这张表格定制的软件，就做出来了。

用了3天，表格格式变了，软件没用了？那就扔了呗。转头做个新的。反正便宜。

软件的最终成本，是电费。

那么，这对程序员意味着什么？意味着失业吗？

不！意味着更大的机会。

第一次工业革命，瓦特改良了蒸汽机。煤炭使用效率大增。很多人预测，煤炭的销量将会因此下降。

但情况截然相反：煤炭销量大增。

为什么？因为成本降低，大家本来用不起煤炭的地方，也开始用了。销量反而大涨。

软件行业也一样。

过去，很多企业用不起软件。尤其是中小企业。

十几个程序员，熬几个月的大夜（别问我是怎么知道的），才弄出一套（不那么合身的）软件。当然贵。

所以，很多企业明明需要吃药，却苦笑说：睡一觉就好了。

但今天，软件这款“药”便宜了。

于是，伤痛会立刻变成需求：先给我来40套。

一个全新的软件范式，正在悄然出现：“情景软件（situational software）。不被成本限制，只被情景驱动。

哎呀。一不小心说了这么多。

飞机怎么还没有落地？加拉帕戈斯，实在是太远了。

但也许，也正是因为那么远，才能独立进化吧。也正是因为大家不注意，才是机会吧。

不说了。就这样吧。

我们这家小破公司，毛病太多。我要去继续写药 … 哦不，我要去继续写软件了。

把下个星期的量，先写足。

用中文。
